### 文本文件IO

这套设备专门处理由可见字符（ASCII/UTF-8）组成的履带，它们对 `\n`（换行）和 `\0`（封条）极其敏感。

| 设备名称 (函数) | 代码操作台示例 | 适用工业场景 | 最小物理自动机机制描述 (底层状态机) |
| :--- | :--- | :--- | :--- |
| **`fgetc`**<br>*(单兵探针)* | `int ch = fgetc(fp);` | 逐字节精细分析、手写极简状态机（如手搓 CSV 解析器）。 | **微型镊子**：探针每次向右滑动 1 个物理格，抓起 1 个字节返回。若履带空了，触发限位器，返回 `EOF` (-1)。 |
| **`fgets`**<br>*(安全挖掘机)* | `fgets(buf, 1024, fp);` | 按行安全读取文本、防御缓冲区溢出攻击。 | **带限位器的机械臂**：从履带连铲带拿填入铁盒。遇到 `\n` 或装满 `容量-1` 时急刹车。**停机必做动作：在末尾强制砸入一块 `\0` 封条。** |
| **`fscanf`**<br>*(模式粉碎机)* | `fscanf(fp, "%d", &num);` | 极其规整的数据提取（如纯数字矩阵），强烈不建议读复杂文本。 | **带吸尘器的锻造炉**：先开启吸尘器清空履带前方的所有空白（空格/换行）；然后咬住连续的有效字符融化重铸，砸进你提供的物理坐标（`&num`）里；遇到下一个空白或格式不符瞬间脱落。 |
| **`fputc`**<br>*(单点点胶机)* | `fputc('A', fp);` | 逐个字符输出、写自定义协议头。 | **单格注塑机**：把 1 个指定的字节直接拍在履带当前位置，履带向前推一格。 |
| **`fputs`**<br>*(高压消防水带)* | `fputs(str, fp);` | 高频、高吞吐量的日志写入、已拼接好的纯文本输出。 | **直通阀门**：顺着给定的内存坐标开始向履带狂喷字符。内部传感器只盯着 `\0`，**一撞见 `\0` 瞬间关阀**，且绝对不把 `\0` 喷出去。无解析损耗。 |
| **`fprintf`**<br>*(五轴数控机床)* | `fprintf(fp, "%d", 42);` | 需将内存里的二进制变量（整数/浮点）翻译成人类可见字符的场景。 | **全套解析流水线**：先扫描格式指令串，匹配到 `%` 后换挡，去内存提取原始变量，放入 ASCII 铸造炉转成字符（如把十六进制的 `0x2A` 转成字符 `'4'` 和 `'2'`），再逐一拍到履带上。能耗极高。 |

### 二进制文件IO

这套设备完全剥夺了“语言学”的意义。它们眼里没有换行，没有空格，没有字符串结束符 `\0`，只有**连续的物理硅片状态（0 和 1）**。

| 设备名称 (函数) | 代码操作台示例 | 适用工业场景 | 最小物理自动机机制描述 (底层状态机) |
| :--- | :--- | :--- | :--- |
| **`fread`**<br>*(重型装载叉车)* | `fread(buf, sizeof(int), 10, fp);` | 游戏引擎加载模型资产、读取图像像素矩阵、恢复系统快照。 | **盲眼宽体铲车**：无视任何数据内容。接受两个参数：单块体积、铲取数量。直接从履带上切下指定体积的物理块，原封不动地砸进内存目标区。**返回成功铲起的“完整物理块数量”**，若不够整块则中途拉闸。 |
| **`fwrite`**<br>*(压路机/拓印机)* | `fwrite(buf, sizeof(int), 10, fp);` | 高性能保存雷达数据、封存物理系统状态、写入视频流帧。 | **内存拓印钢板**：锁定内存中的一片连续区域，像盖章一样，把该区域内所有的 `00` 到 `FF` 的电压状态，一微秒内不加任何修饰地拓印到硬盘履带上。`\0` 对它毫无约束力，统统碾压过去。 |
